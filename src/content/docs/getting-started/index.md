# Getting Started

This section is for your first real hour with Fidan.

## The quick path

1. install the CLI
2. verify the toolchain
3. run a tiny program
4. build a native binary
5. understand when to use `run`, `build`, and `--release`

## What Fidan is trying to feel like

Fidan is meant to be easy to start, but not toy-like once you leave
"hello world". The same CLI, language surface, and docs should still make
sense when you move from editing a script to shipping a native build.
